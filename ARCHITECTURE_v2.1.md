# 千川视频批量去重 v2.1 — 架构设计

## 待修复问题

### Bug 1: 重度去重卡住
根因: FFmpeg 单条超时 + 6操作流水线无超时保护
修复: 每个 FFmpeg 调用加 120s timeout，超时自动跳过并标记 ⚠️

### Bug 2: 进度不透明
根因: 只显示 "处理中..."，不知道卡在哪一步
修复: 显示 "变速 2/6 → 音频变调 3/6..." 实时操作名

### Bug 3: 底部按钮被挤出
根因: 文件列表 canvas 撑满，按钮在 fold 外面
修复: 固定底部栏 pack(side=bottom) before 文件列表

### 优化: 拖拽、表格、预估、导出

## 技术选型
- tkinter + ttk (内置)
- tkinterdnd2 (拖拽支持, pip install tkinterdnd2)
- 去重引擎不变 (6项操作池)

## FFmpeg 安全包装
```python
def run_ffmpeg(cmd, timeout=120):
    r = subprocess.run(cmd, capture_output=True, timeout=timeout)
    if r.returncode != 0:
        raise RuntimeError(f"FFmpeg failed: {r.stderr[:200]}")
```

## 线程回调增强
```
callback("变速 1/6") → UI 显示 "🔄 变速 (1/6)"
callback("色彩微调 2/6") → UI 显示 "🔄 色彩微调 (2/6)"
callback("done", ["变速","色彩","缩放","音频"]) → UI 显示 "✅ 变速,色彩,缩放,音频"
callback("error", "FFmpeg超时") → UI 显示 "❌ 超时"
```

## 时间预估
- 记录每条完成视频的耗时
- 平均耗时 × 剩余条数 = 预估剩余时间
- 显示在进度条旁边

## 窗口固定底部
```python
# 先 pack 底部栏 (side=bottom)，再 pack 可滚动区域 (fill=both, expand=True)
bottom_bar.pack(side="bottom", fill="x")  # 始终可见
treeview.pack(fill="both", expand=True)    # 滚动区域
```

这样无论文件列表多长，开始/停止/导出按钮始终在底部可见。
