# 千川视频工具箱 v3.0 — 架构设计

## 概述

在现有「视频批量去重」功能基础上，新增「短视频下载（去水印）」和「文案提取」两个模块。三大模块集成在同一 tkinter GUI 内，用 Tab 切换。

---

## 一、整体架构

```
千川视频工具箱 v3.0
├── Tab 1: 📹 视频去重 (已有)
├── Tab 2: 📥 视频下载 (新增)
└── Tab 3: 📝 文案提取 (新增)
```

### 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| GUI | tkinter + tkinterdnd2 | 保持现有框架，三Tab切换 |
| 打包 | PyInstaller + Inno Setup | GitHub Actions自动构建EXE |
| 下载引擎 | `f2` (PyPI) | MIT协议，CLI干净，直接pip安装 |
| OCR | PaddleOCR 或 cnocr | 中文OCR，识别视频字幕 |
| 语音转文字 | FunASR 或 faster-whisper | 提取口播文案 |
| 视频处理 | ffmpeg | 音视频分离/格式转换 |

## 二、Tab 2: 视频下载模块

### 2.1 功能

- 输入/粘贴抖音分享链接（批量支持换行分隔）
- 自动去水印下载高清视频
- 同步提取视频标题（desc字段）
- 支持多平台：抖音、TikTok、快手、B站
- 下载进度显示

### 2.2 技术选型

**推荐：`f2` (Johnserf-Seed/TikTokDownload)**

```
pip install f2
f2 dy -u "https://v.douyin.com/xxxxx/"
```

- MIT开源协议，商业友好
- GitHub 8,653 Stars，活跃维护
- CLI单命令下载，输出稳定，易于集成
- 依赖轻（httpx + cryptography），打包后约20MB
- 不支持yt-dlp — 2024年起Douyin提取器持续报错，无法修复

### 2.3 去水印原理

不是图像处理去掉水印叠加层。抖音API返回多个清晰度视频地址，**最高清的那条天生不带水印**。只需按码率排序取最高的 `play_addr` 即可。

```
API返回结构:
 bit_rate = [
   {bit_rate: 1500k, url: "xxx_watermarked.mp4"},   ← 低清，带水印
   {bit_rate: 4500k, url: "xxx_clean.mp4"},         ← 高清，无水印
 ]
```

**结论：零视频处理，纯URL选择。**

### 2.4 稳定性风险

抖音反爬不断升级（a_bogus签名参数），`f2` 作者维护活跃但需跟进。**这是本模块最大的不确定性**——一旦抖音API大改，下载功能可能暂时失效，需要等作者更新。

## 三、Tab 3: 文案提取模块

### 3.1 三种提取方式

| 方式 | 来源 | 准确率 | 速度 | 成本 |
|------|------|--------|------|------|
| A. API标题 | 抖音desc字段 | 100% | 即时 | 免费 |
| B. OCR识别 | 视频画面字幕 | 90-95% | 2-5秒/帧 | 免费 |
| C. 语音转文字 | 视频音频口播 | 85-95% | 10-120秒/条 | 免费 |

### 3.2 方式A: API标题 (已内置)

下载视频时 `f2` 自动返回标题。**零额外开发量**，下载完成直接展示。

### 3.3 方式B: OCR字幕识别

**首选：cnocr（轻量，推荐生产环境）**

```
pip install cnocr
# 模型大小仅 30-50MB，CPU可用
```

```python
from cnocr import CnOcr
ocr = CnOcr()
out = ocr.ocr('frame.jpg')
```

**备选：PaddleOCR（精度略高但体积大）**

```
pip install paddleocr
# 模型约500MB+，需下载paddlepaddle
```

| | cnocr | PaddleOCR |
|---|---|---|
| 中文准确率 | ~90% | ~95% |
| 模型大小 | ~50MB | ~500MB+ |
| 打包体积 | 小 | 大(EXE可能>500MB) |
| 推荐场景 | **生产环境** | GPU服务器 |

**OCR流程：**
```
视频 → ffmpeg抽帧(每2秒) → cnocr逐帧识别 → 合并去重 → TXT输出
```

### 3.4 方式C: 语音转文字

**首选：FunASR（阿里达摩院，中文最优）**

```
pip install funasr
# Paraformer-large 模型 ~1GB，首次自动下载
```

- 中文ASR准确率高于Whisper
- 自动语音活动检测(VAD，跳过静音)
- 自动标点恢复
- MIT协议

**备选：faster-whisper（通用方案）**

```
pip install faster-whisper
```

- CPU速度是原版Whisper的4倍
- large-v3模型2.9GB，中文效果好
- medium模型1.5GB，平衡方案

**语音提取流程：**
```
视频 → ffmpeg提取音频(wav 16kHz) → FunASR推理 → 带标点文案输出
```

### 3.5 模型体积与打包考量

| 组件 | 模型大小 | 是否打包进EXE | 建议 |
|------|---------|-------------|------|
| f2 | 无需模型 | ✅ 打包 | 必须 |
| cnocr | ~50MB | ✅ 打包 | 推荐 |
| PaddleOCR | ~500MB | ❌ 太大 | 不打包 |
| FunASR | ~1GB | ❌ 太大 | 不打包 |
| faster-whisper medium | ~1.5GB | ❌ 太大 | 不打包 |

**策略**：
- **基础版**：只打包 f2 + cnocr（EXE约150MB），覆盖 下载+API标题+OCR字幕
- **高级版**：语音转文字作为**可选下载功能**，首次使用时自动从网上下载模型（提示用户需要联网）

## 七、风险点

| 风险 | 应对 |
|------|------|
| 抖音API反爬升级 | `f2` 作者活跃跟进；如长期不修复可换 Evil0ctal API |
| cnocr 对复杂字幕效果差 | 先做字幕区域裁剪提高准确率；复杂场景降级到FunASR |
| FunASR模型太大不便携 | 不打包进EXE；首次运行时从ModelScope自动下载 |
| EXE打包体积膨胀 | 基础版只含 f2+cnocr；语音功能按需下载模型 |
